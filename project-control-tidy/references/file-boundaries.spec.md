# 控制面文件边界规范

用于裁决 `AGENTS.md`、`README.md`、`justfile` 各自承载什么。

目标：避免把约束、语义、导航、执行入口混写到同一个文件里。

## `AGENTS.md`

作用：承载协作约束。

应包含：

1. 可改范围与禁区。
2. 停止条件与人工接管条件。
3. 规则优先级。
4. 分层 `AGENTS.md` 的继承与覆盖规则。
5. 只有落盘到 `AGENTS.md` 才生效的规则启用机制。
6. 仅保留跨任务都成立的基本工程约束，例如最小盘点、变更边界、验证义务、架构分层与协作立场。
7. 若目录存在结构异常、需求歧义或高风险操作门禁，明确写出暂停、确认与人工接管条件。

详细 guideline 参考 [`agents.guideline.md`](agents.guideline.md)。

不应包含：

- 模块职责介绍
- README 式导航说明
- `justfile` recipe 细节
- 项目实现原理说明

## `README.md`

作用：承载最小概述、职责说明和文档索引。

应包含：

1. 职责与边界。
2. 相关目录、文档、示例或上下游关系的索引。
3. 只有确有必要时才写的最小使用入口或最小阅读入口。

详细 guideline 参考 [`readme.guideline.md`](readme.guideline.md)。

不应包含：

- 协作规则优先级
- 停止条件或接管条件
- 大量命令实现细节
- 需要长期维护的详细背景说明

## `justfile`

作用：承载统一执行入口。

应包含：

1. 对不同语言工具链的统一聚合入口。
2. `libs` / `projects` 的稳定执行入口。
3. 必要且最小的参数说明。

详细 guideline 参考 [`justfile.guideline.md`](justfile.guideline.md)。

不应包含：

- 业务背景介绍
- README 式长说明
- 与当前入口无关的协作规则

## 文件新增规则

只有以下条件同时成立时，才新增控制面文件：

1. 当前路径确实缺少稳定协作所需的最小骨架。
2. 新文件承载的是独立且长期存在的职责。
3. 该职责不能由上层现有控制面自然覆盖。

若不满足，优先修改现有文件，而不是继续铺新文件。
